MSI Z370 게이밍 프로 카본 (feat. 8700K)


커피레이크가 출시한지도 벌써 2달이 넘어가고 있습니다. 출시 초기에 구경도 못할 수준의 공급량, 그리고 말도 안되는 높은 가격이었으나 조금씩 가격이 내려가면서 많은 분들이 커피레이크 업그레이드를 단행 할 것으로 예상됩니다.


그에 따라 커피레이크 CPU + Z370 메인보드를 활용한 오버클럭킹 방법에 대한 소개를 해보려 오버가이드를 작성해보았습니다. 본 게시글에서는 영상을 통해 소개해드린 MSI Z370 게이밍 프로 카본을 활용하여 i7-8700K 오버클럭킹 방법을 소개할 것 입니다. MSI 메인보드 바이오스 기준으로 설명이 진행되지만 타사 바이오스에서도 비슷한 기능이 있는 만큼 내용을 참고하시어 활용하신다면 도움이 되시기라 생각해봅니다.


글 작성에 앞서 오버클럭킹 결과는 CPU 수율, 메인보드 바이오스 완성도, 쿨링 환경 등에 따라 상이하게 다른 결과를 보여줄 수 있습니다. 구매하신 제품의 수율과 쿨링 환경에 따른 배수 및 전압 조절은 필수입니다. 가이드를 참고하셔서 수율에 맞는 오버클럭킹에 성공하시길 기원합니다.







제품 소개


MSI Z370 게이밍 프로 카본 + 인텔 코어i7-8세대 8700K (커피레이크)를 준비하였습니다. 오버클럭킹에 들어가기 전에 패키지 구성, 외형을 간략하게 살펴보도록 하겠습니다.


후면 I/O 쉴드, SLI 브릿지, 메뉴얼, 소프트웨어 DVD 등 다양한 구성품이 동봉되어 있습니다.

이전 시리즈(Z270 게이밍 프로 카본)와 하위 제품인 크레이트 게이밍 등과 비교해보았을 때 레이아웃은 비슷하지만 I/O 및 M.2 커버, 방열판 등 좀 더 심플하고 고급스러운 재질이 느껴지며 상위급 다운 면모를 확인할 수 있습니다.




테스트 시스템


테스트를 위해 아래와 같은 주요부품으로 구성된 PC를 사용하였습니다.


CPU : 인텔 코어i7-8세대 8700K (커피레이크) / RAM : AVEXIR 8G PC4-19200 GREEN x 2 / COOLER : ID-COOLING AURAFLOW 240 RGB BLACK / M/B : MSI Z370 게이밍 프로 카본 / VGA : MSI GTX1070 Ti 게이밍 D5 8GB 트윈프로져6









오버클럭킹 메뉴 소개

바이오스를 진입하면 처음 만나게 되는 EZ MODE 화면입니다.

오버클럭킹 메뉴를 위해서는 ADAVNCED MODE (F7 입력)로 진입하여야합니다.



ADVANCED MODE (F7 입력)에 진입한 모습입니다. 다른 메뉴는 제쳐두고 오버클럭킹 셋팅을 할 수 있는 [ OC ] 메뉴를 바로 확인해보도록 하겠습니다.

– OC Explore Mode

Normal : 기본 메뉴제공 / Expert : 세부메뉴 추가 제공 (권장)

오버클럭킹 메뉴 구성을 선택할 수 있는 기능으로 Normal 구성으로도 오버클럭킹은 가능하지만 상세 설정을 불러올 수 있는 Expert를 권장합니다.

– CPU Ratio Apply Mode

All Core : 모든 코어 배수적용 (권장) / Turbo Core : 터보 코어 배수적용 / Per Core : 각 코어별 개별 배수적용

CPU 작동클럭은 ( 100 x 배수 )로 작동을 하게 되어 있습니다. 그래서 오버클럭킹은 배수(Ratio) 값을 입력하여 작동클럭을 지정하게됩니다. CPU Ratio Apply Mode는 이러한 배수(Ratio) 부여방식을 설정하는 메뉴로 특별한 일이 없는 경우 All Core 를 선택하여 모든 코어에 동일한 배수 설정을 부여합니다. 

* 상황에 따라 코어별로 별도의 배수 설정을 활용할 수 있습니다. 예를들어 3번 코어의 온도가 너무 높은 경우 Per Core를 통해 3-Core 배수만 조금 낮게 지정하여 온도를 낮출 수 있습니다.



– Misc Setting / EIST

기본값 : Enabled / Disabled (권장)

EIST 다른 말로는 스피드 스텝은 전력관리 옵션 중에 하나로 사용량에 따라 가변클럭으로 작동하게 합니다. 작동 클럭이 가변되는 상황에서 안정화를 진행하는 경우 미세하게나마 전압에 영향을 미칠 수 있기 때문에 해당 기능을 끄고 안정화를 진행할 것을 권장하지만 코어인가 방식 및 사용자 성향에 따라 켜두는 경우도 종종 있습니다.


* 코어전압 인가 방식에 따라 켜두는 것이 좋은 경우도 존재합니다. (Adaptive , Offset 의 경우 켜는 것을 권장)

– CPU Ratio Offset When Running AVX

Auto : 자동적용 / 직접 값입력 : 권장값 -3 ~ -5 (추천 -5)

커피레이크 오버에 있어서 많이 활용을 하게될 기능으로 AVX 명령어를 사용하는 경우 입력된 값만큼의 배수를 하락시켜 과부하를 줄여주는 역할을 합니다. 예를 들어 CPU 50배수 (5.0GHz) 설정에서 AVX -5로 설정하는 경우 LINX 테스트에서 -5배수 하락된 45배수 (4.5GHz)로 작동을 하게됩니다. AVX 명령어를 사용하지않는 경우 AVX -5에서도 정상적인 50배수 (5.0GHz)로 작동하게 됩니다. 


* AVX 명령어 작동에도 50배수로 동일하게 작동하도록 설정하는 것이 좋은 성능을 기대할 수 있지만 AVX 명령어를 사용할 때 급격하게 온도가 상승하는 만큼 사용타입에 따라 해당 기능을 활용하는 것을 권장합니다. AVX 0 설정으로 사용하고 싶다면 CPU IHS 튜닝(뚜따)를 추천드립니다. 노뚜따에서는 AVX Offset을 활용하시기 바랍니다.

– Ring Ratio

Auto : 자동적용 / 직접 값입력 : 권장값 -3 ~ -5 (추천 -5)

CPU 캐쉬 배수(Ratio)를 설정하는 곳으로 일반적으로 작동배수 -5 정도로 셋팅을 권장합니다.


* 캐쉬 배수를 높일 수록 성능은 좋아지지만 CPU 코어전압 상승 및 안정화 난이도가 상승합니다.

– CPU Base Clock (MHz)

기본값 : 100 (권장)

앞서 배수설명에서 적었듯이 CPU 작동클럭은 내부클럭(BLCK) 100 x 배수로 작동하게 되는데 해당 메뉴를 통해 내부클럭 100에서 좀 더 높게 설정이 가능합니다. 다만, 내부클럭을 상승시키는 경우 VGA, SSD, HDD 등 주변 부품에 영향을 끼칠 수 있기에 기본값 100으로 사용하기를 권장합니다.

* 배수언락 (K버전)이 아닌 일반 CPU에서 작동클럭을 높이기 위해 사용되는 경우가 있습니다. (i5-8400 / i7-8700 등)

– Extreme Memory Profile(X.M.P)

기본값 : Disabled / XMP 메모리 사용시 : Enabled

사전 테스트를 거쳐 판매되는 오버클럭킹 메모리 (G.Skill , Corsair , TeamGroup , Kingston 등)에 설정된 X.M.P 프로파일을 불러올 수 있는 기능으로 해당 제품들을 사용하는 경우 Enabled 로 설정하시기 바랍니다.



– DRAM Reference Clock

기본값 : Auto(권장) / 100 / 133

메모리 레퍼런스 클럭을 설정하는 기능으로 특별한 경우가 아니라면 Auto를 권장합니다.



– DRAM Frequency

기본값 : Auto(권장) / 

메모리 오버클럭킹을 할 수 있는 기능으로 각 제품의 수율에 따라 작동클럭을 선택하여 안정화를 진행하게 됩니다. 하지만 Auto를 권장하는 이유는 메모리 오버클럭킹의 경우 CPU오버와 함께 진행하는 경우 CPU + RAM 두가지에서 어떤 부분이 오버가 실패하는지 초심자의 경우 파악하기 쉽지않으며, 추가적으로 메모리 오버를 진행하는 경우 CPU 코어전압의 영향도 있기에 난이도가 상승하므로 메모리 오버는 차후에 좀 더 오버스킬을 쌓은 후 진행하는 것을 추천합니다.



– Advanced DRAM Configuration

메모리 작동 타이밍을 세부적으로 설정할 수 있는 메뉴이나 전문가가 아닌 이상 해당 메뉴는 기본값을 그대로 사용하는 것을 권장합니다. 본 가이드는 초심자를 기준으로 하고 있기에 앞서 언급한 DRAM Reference Clock / Frequency 와 함께 기본값 그대로 사용하는 것을 권장합니다.


– DigiALL Power (CPU LoadLine Calibration Control / LLC)

기본값 : Auto / Mode 4 (권장)

일명 LLC라 불리우는 기능으로 CPU 로드 상황에서 전압강하를 막아주어 전압부족으로 프리징되는 현상을 방지하는 기능입니다. 오버클럭킹 타겟이 높을 수록 중요하게 다뤄야 할 기능이며 LLC 옵션이 얼마나 세분화 되어있고 제대로 작동하는지에 따라 메인보드 선택에 중요한 구매요소가 될 수 있습니다. 오버클럭킹에 중요한 기능인 만큼 LLC를 잘 다룰수록 오버 성공률을 높일 수 있고 코어전압과 함께 온도관리의 핵심이라 할 수 있습니다.


* Mode 4를 권장하는 이유는 코어전압을 강하없이 지탱하는 칼전압 수준의 기능을 보여주기 때문이며, 이 특징은 MSI Z370 메인보드에만 해당합니다. LLC 옵션 스타일은 각 제조사마다 특색이 있기에 같은 제조사 사용자들의 오버값을 참고하시기 바랍니다.



– CPU Core/GT Voltage Mode

기본값 : Auto / Adaptive Mode / Override Mode(권장) / Offset Mode

/ Adaptive + Offset Mode / Override + Offset Mode

CPU/iGPU 전압을 설정하는 기능으로 오버클럭킹에서 LLC와 함께 가장 중요한 부분입니다. CPU 수율이란 얘기를 하는 부분이 바로 코어전압 때문이며 온도와 직결되기에 좀 더 낮은 전압을 인가하는 것을 선호하며, 이러한 전압값에 의해 고수율 / 뿔딱 CPU가 발생하며 수율뽑기가 존재하는 것입니다.

MSI 메인보드에는 코어전압 인가방식이 5가지로 상당히 디테일한 편인데 가장 손쉬운 Override Mode (Fixed) 기능을 권장합니다. Override Mode는 항상 설정한 전압이 적용되는 Fixed 방식으로 가장 쉬운 전압인가 방식이며 많은 사용자가 선호하는 방식입니다. 기능을 선택하면 바로 아래에 CPU Core Voltage 값을 입력할 수 있게되며 이 곳에 1.00 ~ 1.520V 까지 수동 전압을 입력할 수 있습니다. 일반적으로 1.5V이상의 한계치를 가지고 있지만 1.45V 이상의 전압은 인가하지 않습니다. 


그 외에도 SA / IO / PLL OC / PLL SFR / RING PLL / SA PLL / MC PLL / CPU ST PLL / DRAM Voltage 등 다양한 전압인가 기능이 있지만 특별한 경우가 아니라면 기본값(Auto)를 권장합니다. 본 가이드에서도 초심자를 기준으로 한 만큼 해당 기능들은 활용하지 않습니다.

 CPU Core Voltage [ 1.360 ]  – CPU 수율에 따라 – / + 적용하시기 바랍니다.

일반적으로 8700K 노뚜따, 중급형 4~5만원대 공랭 쿨러를 장착한 경우 [email protected] (AVX -3) 셋팅을 추천합니다. 대표적인 쿨러로는 ‘써모랩 트리니티’가 해당됩니다. 노뚜따 상태에서 50배수 (AVX -5) 설정 또는 48배수 (AVX 0) 설정으로 클럭을 올려야한 다면 좋은 성능에 쿨러가 필요하며, 이에 해당하는 제품은 공랭에서는 ‘써멀라이트 마쵸 RT / 녹투아 D15’ 급의 제품, 일체형 수냉에서는 ‘NZXT X52 / Corsair H100i’ 급의 제품이 필요합니다.


이러한 고급형 쿨러 강제현상은 인텔의 똥서멀에 의한 것으로 최상위 쿨러를 사용함에 있어서도 높은 발열은 감수해야할 부분입니다. 특히 AVX 클럭을 높이면 높일수록 발열 증가 현상이 두드러지기 때문에 AVX Offset을 통해 배수하락 기능을 활용하는 것이 이로우며, 해당 옵션을 사용하고 싶지 않다면 반드시 IHS 튜닝(뚜따)를 진행해서 온도를 잡아야합니다.






오버클럭킹 안정화

안정화 테스트란 것은 오버클럭킹을 적용 후 CPU / RAM에 인위적인 100% 로드상황을 부여해서 블루스크린, 프리징, 재부팅 등의 현상이 발생하는지, 작동 온도와 쿨링 소음은 어떠한지를 체크하는 과정으로 테스트 과정에서 오류상황이 발생하는 경우 오버클럭킹 셋팅 값을 조정하며 진행합니다.


테스트에 주로 사용되는 프로그램은 아래와 같습니다.

CPU-Z / RealTemp / LinX 0.8.0 / Prime95 / RealBench / AIDA64 / HWINFO64 / HW Monitor 등

이번 가이드에서는 CPU-Z / RealTemp / LinX / AIDA64 를 사용하여 안정화 방법을 소개합니다.

테스트에 앞서 클럭고정을 위해 전원관리 옵션을 균형 → 고성능으로 선택합니다. (안정화가 끝난 후에는 전력관리를 위해 균형으로 다시 조정하시기 바랍니다.)




CPU-Z를 통해 오버클럭킹이 제대로 적용되었는지 확인 할 수 있습니다. 스샷을 통해 확인할 수 있듯이 50배수가 적용된 상태로 Core Speed / Multiplier 를 확인할 수 있습니다.


LinX / AIDA 64 테스트에 앞서 온도 확인을 위해 Real Temp GT를 실행합니다. 해당 프로그램을 통해 CPU 온도를 확인할 수 있으며 최대 온도 / Thermal Status를 확인할 수 있습니다. 100도를 넘기는 경우 Thermal Status에 LOG 또는 HOT이 표기되며 이 경우 블루스크린, 프리징이 없더라도 오버클럭킹은 실패한 것입니다. 테스트 과정에서 LOG / HOT이 발생하는 경우 전압조정 및 배수조정을 통해 온도를 100도 미만으로 낮추어야 합니다.



LinX 0.8.0을 통해 CPU / 메모리 과부하 연산을 진행하며 연산 과정에서 AVX 테스트를 함께 진행하게 됩니다. 보통 20회 이상을 통과하는 것이 목표이며 시스템 메모리에 따라 테스트 메모리 사이즈를 선택하여 진행합니다. 일반적으로 시스템 메모리의 7~80%를 설정하여 테스트를 진행합니다.


* 8GB 기준 : 6,144MB 권장 / 16GB 기준 : 12,288MB 권장 / 32GB 기준 : 24,576MB 권장


#AVX Offset을 적용한 경우에만 테스트 진행

* AVX Offset 0으로 오버클럭킹 설정을 한 경우에는 LinX 테스트만 진행하여도 무방합니다.


AIDA 64에는 System Stablility Test를 통해 CPU / 메모리 과부하 테스트가 가능하며 해당 테스트에서는 AVX 명령어는 사용되지 않습니다. 일반적으로 1~2시간 정도 테스트를 진행하며 오류가 없는 경우 안정화로 판단합니다.


LinX 테스트만 하는 것이 아니라 AIDA 64 안정화를 진행하는 이유는 오버클럭킹 설정에서 AVX Offset 기능을 사용한 경우를 고려해서 LinX 테스트시 AVX가 작동할 때 설정된 값만큼 배수가 하락하기 때문에 (50배수 AVX -5인경우 45배수로 작동) AVX를 사용하지 않을 때 50배수의 안정화를 따로 진행하는 것입니다. (LinX = :45배수 / AIDA 64 : 50배수를 테스트)




# 블루스크린, 프리징, 재부팅 해결방안


앞서 소개한 프로그램을 통해 안정화 테스트를 진행함에 있어 블루스크린, 프리징, 재부팅 등의 현상이 발생한다면 오버클럭킹 설정값의 조정이 필요합니다.


가장 먼저 테스트시 최대 온도를 체크하여,

100도 미만인 경우 : – CPU 전압 추가 -를 통해 코어 전압을 안정적으로 공급해보아야합니다. 코어 전압은 0.010 ~ 0.020v 수준으로 추가하며 테스트 진행하여 오작동이 없는 전압을 찾으시기 바랍니다.


100도 이상인 경우 : 좀 더 다양한 확인이 필요합니다.

우선 온도를 낮추기 위해서는 – 전압 / 배수 다이어트 – 를 진행해야 하는데, 50배수 테스트를 예를 들면 50배수 상태에서 온도만 높다면 전압 다이어트를 통해 온도를 낮출수 있습니다.


그러나 높은 온도 + 블루스크린(재부팅) 등이 발생한다면 코어 전압을 추가해야 합니다. 다만, 이미 CPU 온도가 매우 높기에 전압 추가는 불가능한 상태이기에 아쉽지만 작동클럭 조정(배수)를 낮춰야 합니다.(50배수 → 48배수와 같이 다운클럭)


그럼에도, 50배수 오버를 원한다면 추가된 코어 전압에서도 안정적인 CPU 온도 확보를 해야하기에 현재 사용하고 있는 CPU 쿨러 보다 상위급 제품으로 교체하거나, IHS 튜닝(뚜따)를 해서 온도를 낮추어야 합니다. (또는, CPU를 장터에 정리하고 새로 구입해서 수율 뽑기를 시도합니다. – 비추)



지금까지 MSI Z370 게이밍 프로 카본을 활용한 인텔 8세대 커피레이크 8700K 오버클럭킹 방법에 대하여 소개하였습니다. 가이드에서는 Z370 게이밍 프로 카본으로 설명하였지만, MSI Z370 메인보드라면 바이오스 구조가 같기에 해당 제품이외에 전체 제품에 참고가 가능하시리라 생각됩니다.


MSI Z370 시리즈는 사용자 표본이 얼마없어 보이지만 생각보다 괜찮은 메인보드라 생각합니다. 타사에 비해 바이오스가 늦게 공개되는 점은 단점으로 생각되지만 초기 바이오스가 크게 문제가 있는 것은 아니기에 오버클럭킹을 적용하기에 문제는 없다고 판단됩니다. (그래도 바이오스는 좀 빨리 빨리 공개하자…일해라 므시!!)


MSI Z370 제품군을 사용하시는 분들게 도움이 되셨기를 바라며 MSI 제품외에도 타사 제품에도 명칭은 다르지만 같은 기능이 있다면 비슷한 값을 넣은 후 조절하실 수 있으리라 생각합니다. 초심자를 위한 오버 가이드를 작성해본다고 했는데 도움이 되셨을지 모르겠습니다. 이 글을 통해 조금이라도 좋은 오버결과 있으시길 바라며 글을 마칩니다.
