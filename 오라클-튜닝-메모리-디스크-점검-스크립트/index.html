MEMORY 및 DISK 점검 및 튜닝


대상 별 점검값 정상범위 및 조치

대 상

 점검 수치 점상범위 

 조 치

 Library Cache

 Miss Ratio(%) < 1

 SHARED_POOL_SIZE를 늘려 줌(Default 3.5M)

 Data  Dictionary Cache

 Miss Ratio(%) < 9.8

  SHARED_POOL_SIZE를 늘려 줌 

 DB Buffer Cache

 Hit Ratio(%) > 70

 DB_BLOCK_BUFFER를 늘려 줌

 Datafile

 Physical Read, Physical Write

 PHVRDS와 PHYWRTS 값이 큰 Datafile을 다른 디스크에 분산 저장

 Datafile-Sort

 sorts(memory) > sorts(disk)

 1. SORT_AREA_SIZE를 늘려 줌

 2. SORT_AREA_SIZE와 SORT_AREA_RETAINED_SIZE를 

    적절히 조정해서 DISK I/O를 줄여 줌

    (OLTP환경에서 SORT_AREA_SIZE가 두배)

 Rollback Segment

 Miss Ratio(%) < 1

 Rollback Segment의 갯수를 늘려 줌

 Redo Log File

 Redo Log Request = 0

 0근처가 아니면 LOG_BUFFER를 늘려 줌

 Free List_Buffer Cache

 Wait Radio(*) < 1 

  – (제 생각엔 아마도 단순하게 getLatch()를 위한 SPIN_COUNT를 늘려주거나 sleep()타임을 줄여주면 되지 않을까요?)

  select
distinct d.file_id               file#,

  d.tablespace_name                 ts_name,

d.bytes /1024 / 1024               MB,

d.bytes / 8192                     total_blocks,

  sum(e.blocks)                      used_blocks,

to_char( nvl( round( sum(e.blocks)/(d.bytes/8192), 4),0) *100,’09.99′) ||
‘ %’ pct_used

from   dba_extents 
e, dba_data_files  d

  where 
d.file_id = e.file_id(+)

  group by 
d.file_id , d.tablespace_name , d.bytes

  order by 1,2 ;
