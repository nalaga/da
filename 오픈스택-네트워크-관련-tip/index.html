# 오픈스택 네트워크 관련 Tip 입니다
인스턴스에 지정된 IP를 할당하여 배포하기
인스턴스 vNIC 1개에 2개 이상의 IP를 사용하기 = Secondary IP 할당
인스턴스 vNIC 1개에 Any MAC/Any IP 허용하기
인스턴스에 IP 할당되는 과정 확인 : DHCP 패킷 분석
# 테스트 환경
Host OS : CentOS7
Openstack 버전 : Ocata
– provider network : 192.168.254.0/24
– selfservice network : 10.0.0.0/24

# 인스턴스에 지정된 IP를 할당하여 배포하기
사례 : 고정된 서버의IP나 혹은 주요VM의 경우 IP를 지정해서 배포가 필요 시
## Network – Subnet – Port(IP) 확인
openstack network list
openstack subnet list
openstack port list

## 수동으로 Port(=지정된 IP) 자원을 생성하기 : 지정된 IP생성은 horizon 에서는 불가능
openstack port-create –network ‘네트워크ID’ –fixed-ip subnet_id=’서브넷ID’,ip_address=’지정된IP’ ‘포트이름’
생성하고 상태는 DOWN 이다. 이유는 아직 Instance에 할당하지 않았기 때문이다
openstack port create –network b527f7d7-b252-4e3e-ab7a-33dac5279c2f –fixed-ip subnet=d788347c-2c80-4ade-b571-a76c375fbf7f,ip-address=10.0.0.50 testport1
openstack port list

호라이즌 대쉬보드에서 생성된 Port(=IP) 정보 확인 : 네트워크 -> 포트

## 인스턴스 배포 시 위에서 생성된 지정된 IP를 할당하기
‘네트워크’를 선택하는 것이 아니고 ‘네트워크 포트’를 선택 후 미리 생성해놓은 Port(=IP)를 할당하면 됨

## 할당된 IP를 확인
openstack server list

## 생성된 포트(=IP) 삭제 시
openstack port delete “포트이름”
# 인스턴스 vNIC 1개에 2개 이상의 IP를 사용하기 = Secondary IP 할당
사례 : 이중화 동작(클러스터링 등)이나 Proxy 처리(L4, NAT 등) 및 트래픽제너레이터 – 1개 NIC에 여러IP가 필요함

## 기본 상태 확인
(인스턴스) ifconfig

(오픈스택 컴퓨트노드) ebtables -L
기본적으로 지정된 ip-mac 만 허용됨


(오픈스택 컨트롤러노드) openstack port show “포트이름 혹은 포트ID”
alowwed_address_pairs 에 현재는 아무IP값도 없음


(오픈스택 컨트롤러노드) horizon 대쉬보드 : 네트워크 – 포트 – 포트선택 – 허용된 주소쌍
추가로 허용된 IP는 현재 없는 상태임(=허용된 주소 쌍)

## 인스턴스내의 1개 NIC에 여러개의 IP를 할당하기
(인스턴스)
ifconfig eth0:51 10.0.0.51 netmask 255.255.255.0 up
ifconfig eth0:52 10.0.0.52 netmask 255.255.255.0 up
ifconfig eth0:53 10.0.0.53 netmask 255.255.255.0 up
ifconfig

## 다른 인스턴스에서 ping 실행 시 실패함~
## 해당 포트에 여러개의 IP가 허용될수 있게 포트정보를 업데이트(=set)하자
openstack port set –allowed-address ip-address=’추가할IP’ ‘포트이름’
openstack port set –allowed-address ip-address=10.0.0.51 testport1
openstack port set –allowed-address ip-address=10.0.0.52 testport1
openstack port set –allowed-address ip-address=10.0.0.53 testport1
(오픈스택 컴퓨트노드) ebtables -L
추가한 IP-arp 가 허용될 수 있게 업데이트 된 것을 확인 할 수 있다


(오픈스택 컨트롤러노드) openstack port show “포트이름 혹은 포트ID”
3개의 IP가 허용된것을 알 수 있다


(오픈스택 컨트롤러노드) horizon 대쉬보드 : 네트워크 – 포트 – 포트선택 – 허용된 주소쌍
CLI에서 확인한 내용을 대쉬보드에서도 확인 할 수 있음

## 다른 인스턴스에서 ping 성공 확인

# 인스턴스 vNIC 1개에 Any MAC/Any IP 허용하기
사례 : 트래픽제너레이터(VM타입)의 경우 Source IP를 여러개를 사용 시 아에 Any MAC/Any IP를 사용이 필요함
사례 : vRouter(L3 Agent)의 Port(=IP) 와 dhcp의 Port(=IP)의 경우
openstack port set –no-allowed-address testport1
## 아래 처럼 port-security 를 disable 해버리자
openstack port set –no-security-group –disable-port-security testport1
openstack port show testport1


## 아래 ebtables 에서도 정책이 사라짐
ebtables -L
## 이제 instance 에 any ip/any mac 을 넣고 테스트 할 수 있음
## vRouter(L3 Agent)의 Port(=IP) 와 dhcp의 Port(=IP) 확인
1개의 내부네트워크에 포트 정보 확인 -> 이중 GW역활인 10.0.0.1을 클릭

 

라우터 포트(=IP)의 경우 Port Security 가 False로 되어 있음을 확인

# 인스턴스에 IP 할당되는 과정 확인 : DHCP 패킷 분석
## 확인 방법
Instance 에 매핑된 tapXX 인터페이스에 패킷 캡쳐 진행 : tshark -nni tapXX -w /tmp/test.pcap
## Instance 의 vNIC에서 DHCP 서버(데몬)에 요청하는 파라미터들 = DHCP Request 패킷
MTU 정보, 서브넷 정보, 라우팅 정보, NTP 서버 정보, DNS 서버IP정보 등등

## DHCP서버가 전달해준 정보 = DHCP ACK 패킷
인스턴스의 할당 IP정보, 인스턴스 hostname, 라우터IP정보, DNS 서버IP정보, MTU 크기정보 등
